# 想法备忘：zero-desktop 两个待办功能（对话总结 + 截图自动化）

> 桌面端（zero-desktop）后续想加的两个功能，先固化下来，待有时间再细化设计与实现。
> 记录于 2026-06-15。当前仅为需求记录，**未设计、未实现**。

## 功能一：对话消息总结

### 需求
允许用户把**部分会话内容**复制粘贴进桌面程序，由程序自动梳理并生成**总结**。

### 初步方向（待细化）
- 输入：一段粘贴进来的会话文本（多条消息、可能带角色/时间戳，格式不规整）。
- 处理：调 LLM（复用现有 `LLM_BASE_URL` / `LLM_MODEL` 那套 GB10 vLLM 配置）做梳理 + 总结。
- 输出：结构化总结（要点 / 结论 / 待办等），可复制带走。
- 待定问题：
  - 是否需要先做「会话解析/分角色」预处理，还是整段丢给 LLM。
  - prompt 模板与版本管理（参考 douyin refine 的 `PROMPT_VERSION` / `prompt_hash` 约定）。
  - 是否落盘 / 历史记录，还是一次性。

## 功能二：截图流程自动化

### 现状痛点（手动多步）
1. 按 Print Screen 截全屏；
2. 用微信截图工具选取区域；
3. 绘制红框 / 箭头标注；
4. 再手动复制。

整套流程繁琐，希望由桌面程序**一键自动化**。

### 初步方向（待细化）
- 目标：区域截图 + 标注（红框/箭头）+ 复制到剪贴板，尽量一步到位。
- 待定问题：
  - 标注是程序内置一个轻量标注界面，还是仍借助系统/第三方工具。
  - 区域选取交互（拖拽选框）的实现方式（Tauri 端 + 前端 overlay？平台相关）。
  - 平台：当前是 Windows 11 优先。
  - 与功能一是否联动（截图后直接进总结/记录流程）。

## 状态
两项均为**需求阶段**，优先级低于当前主线（音频清洗 / net-policy 等）。后续动手前再补设计文档。
