# 想法备忘：车载暗电流故障录波仪

> 一个与 toolkit 主线无关的个人硬件想法，先固化下来，后续有兴趣再聊。
> 记录于 2026-06-15。

## 背景问题
旧车出现**间歇性漏电（寄生电流 / 暗电流过大，parasitic draw）**，故障随机、无规律，
修理厂当场难以复现，因此排查困难。

## 核心想法
把一个小型电流监测设备**长期置于车内持续运行**，记录车内线路电流并带时间戳，
**待故障再次发生后通过查阅历史日志精准定位故障根源**——
即把诊断从「现场实时抓取」转向「持续记录 + 事后回溯」。

## 可行性结论
**可行，且切中现成产品的真空。** 方向正确：实时检测产品的盲区在于要求「人在现场 + 故障正好发生」，
而间歇故障不配合。持续记录 + 事后回溯是工业故障录波、飞机黑匣子的同一逻辑。

设备的本质定位应当是：
> 别人做的是**电流表**（此刻多少），我们要做的是**故障录波仪**（故障那一刻前后发生了什么）。

定位明确后，**电流测得多精准反而是次要的**，设计重点转移到下面几条。

## 关键设计要点
1. **分层监测，别一上来测多路线路**
   - 第一层（必做）：只在**电瓶负极总线**串电流 + 电压传感器，测整车静态电流
     （正常约 20–50mA，漏电时熄火后回不落）。单点即可定位 80% 的嫌疑回路。
   - 第二层（按需）：对锁定的嫌疑回路再加钳形/分流传感器精确定位。

2. **环形缓冲 + 异常触发锁存（方案灵魂）**
   - 不能全程高频记录，存储会爆。平时低频滚动（~1Hz），超阈值/突变时切高频（1–10kHz）。
   - **关键：保留故障发生前几秒的数据**（环形缓冲）——漏电源往往在被触发那一刻露出马脚，
     「事前几秒」比「事后」更值钱。类比汽车 EDR / 黑匣子碰撞锁存。

3. **触发逻辑**：用静态电流基线 + 偏差触发；宁可多记误触发，也别漏真故障，事后再筛。

4. **强时间戳 + 事件关联**：记录带精确时间戳、电压、电流、温度，最好蹭一路车辆状态
   （车门、点火、最后唤醒模块）。事后是「对时间线」而非盯孤立曲线猜。

5. **设备自身要能熬数周**：超低自耗（否则它自己成新漏电源）、掉电不丢日志、抗车规环境
   （温度、振动、电压尖峰 / load dump）。

## 供电：倾向自带电源
对「常驻数周抓随机故障」，**自带电源是更优架构**，一次性解决「设备自耗污染读数 + 熬干车电瓶」两大风险：
- 配**霍尔式（非接触）电流传感器**可与车电系统完全隔离，接线简单、抗干扰。
- 掉电独立：车电压骤降那一刻记录仪照常工作存盘。

**核心待解难点**：自供电时不能让 MCU 睡死，否则随机故障来了没人记。需要一路
**常开低功耗看门狗**盯电流、超阈值硬件中断唤醒。矛盾在于常见霍尔（ACS712）自耗 ~10mA 撑不久。
候选解：低功耗霍尔/磁通门（待机几百 µA）、或分流 + 低功耗前端 + 模拟比较器、或周期性快速采样。

**续航估算**：平均 1–3mA 时，一颗 18650（3000mAh）约撑 1–2 个月，够等一次随机故障。

**折中：混合供电** —— 车跑时从车上涓流充电，熄火停车时切自带电池。既不啃静态电瓶又免人工充电，
代价是充电管理电路略复杂。

## 选型骨架（DIY 最小方案）
- MCU：ESP32（自带 WiFi/BLE）或 STM32 + 4G
- 采样：INA226 ×1（总线，分流方案，直流 mA 分辨率够）；隔离/大电流用低功耗霍尔
- 存储：SD / SPI Flash 本地缓冲（联网不可靠，离线先存是底线）
- 上传：4G/Cat-1 模块或回家进 WiFi 机会上传（别指望实时云端）
- 云端：时序数据库（InfluxDB）+ 看板，对齐时间轴看尖峰

## 现成产品调研（为什么还得 DIY）
**没有一个"自带电池、塞车里、自动盯着抓随机漏电、事后看日志"的现成消费产品**——
这个完整形态（自供电 + 常驻 + 触发录波）市面空白，正是本想法的价值所在。
零件级别都有成品，看能接受几分妥协：

| 类别 | 代表 | 局限 |
|---|---|---|
| 带电池的电流数据记录仪（工业/计量） | HOBO(Onset)、Lascar EL、T&D、福禄克电能记录仪 | 多为**交流市电**设计，直流 mA 级暗电流未必测得了；定时记录、**无异常触发锁存** |
| 汽车专用示波诊断 | **PicoScope 汽车版 + 电流钳** | 最专业、能长时间记录回放抓间歇尖峰，但**需挂笔记本**，非自供电盒子 |
| 蓝牙电瓶监测仪 | BM2、Ancel BM300、Topdon BT 系列 | 能长期挂、手机看历史，但**几乎只记电压不记电流**，且吃车电 |
| OBD 记录器 | OBDLink MX+/LX、Carista、ELM327 | 读 CAN 上报数据，非物理电流；可作事件关联那一路 |
| 实验室功耗分析仪 | Otii Arc、Joulescope、Monsoon | µA 级精度带时间戳，但非车载常驻、贵、不防车规 |

**实话**：想零 DIY 买来就用——现成品没有正好对症的，最接近是"带电池的工业电流记录仪"，
需自行验证直流量程/分辨率/是否支持触发，大概率将就。想真正实现设想形态——只能 DIY / 半 DIY。
